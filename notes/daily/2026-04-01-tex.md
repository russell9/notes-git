# Контекст

Задача: спланировать интеграцию TeX в рабочий процесс подготовки материалов, чтобы ускорить верстку формул, повысить единообразие документов и упростить экспорт в PDF/Docx.

# Цель на сегодня

- Определить минимальный рабочий стек (редактор, сборка, шаблон).
- Описать этапы внедрения без остановки текущей работы.
- Подготовить чеклист для первого пилотного документа.

# Основное

- Формат исходников: `.tex` для больших документов, Markdown + TeX для заметок с формулами.
- Базовый движок: `pdflatex` (старт), при необходимости переход на `xelatex` для расширенной поддержки шрифтов.
- Структура проекта:
  - `tex/templates/` - шаблоны
  - `tex/shared/` - общие стили и макросы
  - `tex/projects/<name>/` - документы по проектам
- Минимальный шаблон должен включать:
  - титульную часть
  - подключение пакетов для математики
  - единый стиль заголовков, таблиц, рисунков
- Управление ссылками и библиографией:
  - на старте встроенные ссылки
  - затем добавить `biblatex` + `biber` для автоматизации
  - рассмотреть возможности интеграции с другими программами библиографии

# Риски и ограничения

- Различия в окружениях на разных ПК (пакеты TeX, версии).
- Время на первичную настройку шаблонов.
- Конвертация сложного контента из старых `docx` может быть ручной.

# Критерии готовности

- Документ собирается одной командой без ручных правок.
- Формулы и таблицы выглядят единообразно.
- Новый документ создается из шаблона меньше чем за 5 минут.
